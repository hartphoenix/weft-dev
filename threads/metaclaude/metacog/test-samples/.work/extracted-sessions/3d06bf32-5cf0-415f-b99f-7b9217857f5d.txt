[U5 23:08] i keep ending up with persistent localhost servers even when i think i've killed the terminals they live in. how do i see which ones are still running and optionally kill them?
[A22 23:08] List everything listening on localhost ports:

```bash
lsof -iTCP -sTCP:LISTEN -nP
```

That'll show the process name, PID, and port. To kill a specific one:

```bash
kill <PID>
```

Or if you want to nuke everything on a specific port (e.g. 5173):

```bash
lsof -ti:5173 | xargs kill
```

The persistent servers are probably orphaned Vite/node processes that survived their terminal closing. VSCode terminals in particular can leave these behind.
[U6 23:09] those are gonna be tough to remember. is there a utility on mac that would make this more accessible?
[A24 23:09] Easiest option: add a couple aliases to your `~/.zshrc`:

```bash
alias ports='lsof -iTCP -sTCP:LISTEN -nP'
alias killport='f() { lsof -ti:$1 | xargs kill; }; f'
```

Then it's just:

```bash
ports              # see everything listening
killport 5173      # kill whatever's on that port
```

If you'd prefer a GUI, **Sloth** is a free open-source Mac app that shows all open connections/ports with a nice interface — you can find and kill processes visually. Install with `brew install --cask sloth`.
