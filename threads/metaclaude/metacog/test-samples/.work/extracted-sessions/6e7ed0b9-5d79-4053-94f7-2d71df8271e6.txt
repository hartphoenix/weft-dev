## Design decisions + rationales

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Model abbreviations | SL/ML/LL/MCH/MCO display, --sl etc. flags | Short, scannable in status line. Lowercase flags match shell convention. Abbreviations encode backend class (local vs API) at a glance. |
| Status line model tag | Abbreviation replaces "MC" | The active model is the single most important piece of state during empirical comparison. "Which model am I running?" should be answerable without leaving the terminal. |
| Instruction passing syntax | `+[text]` appends, `-[text]` replaces, `-[]` clears | Append vs replace are both useful. `+` and `-` are minimal, unambiguous prefixes. Square brackets visually delimit the instruction and don't conflict with existing flag syntax. |
| Instruction persistence | File persists across turns until explicitly cleared or metaclaude disabled | Instructions are session-level directives (e.g. "this is a lesson on X"), not per-turn. Persisting avoids re-stating every turn. Cleanup on disable prevents stale state across sessions. |
| Instruction delivery | Appended to system prompt as labeled section | System prompt is the natural home for behavioral directives. Labeled section ("Session instruction (from the user)") makes the source clear to the meta-agent and keeps it separable from the core prompt. |
| Thinking tag handling | Log full response; strip before [no comment] check + injection | The thinking chain is valuable data for empirical comparison (was the model's reasoning good even when it chose silence?). But Builder Claude should never see meta-agent internals — only clean observation text. Two-path design: log path preserves everything, injection path strips. |
| Thinking tag stripping | perl -0777 with non-greedy match | `sed` can't handle multiline tag removal reliably. `perl -0777` slurps the full string and `.*?` matches non-greedily across newlines. Available on macOS by default. |
| Backend routing | `backend` field in model config JSON | Clean separation: toggle.sh writes the config once, observer.sh reads and routes. No hardcoded model→backend mapping in observer.sh. Adding a new backend (e.g. Ollama) means adding one case in toggle.sh and one elif in observer.sh. |
| Backward compatibility | Default to MCH/haiku if no model file | Existing installations with no model file continue to work exactly as before. Zero-breakage upgrade path. |
| LM Studio timeout | 30s max-time on curl | Generous for local inference (~1-3s expected). Prevents hanging hooks if LM Studio is wedged. The hook runner has its own timeout, but this is defense in depth. |
| Hot-swap (formerly deferred) | Now resolved: flag-based via toggle.sh | The "hot-swapping models" item in PRD Decisions Deferred is now resolved by this design. Status line updates immediately; next observer invocation uses the new model. |

### 3d. Response parsing — two-path (log vs inject)

```bash
FULL_RESPONSE=""    # full text including thinking tags (for logging)
CLEAN_RESPONSE=""   # stripped text (for [no comment] check + injection)
INFERENCE_META="{}"

if [ -n "$RAW_RESPONSE" ]; then
    if [ "$BACKEND" = "lmstudio" ]; then
        FULL_RESPONSE=$(echo "$RAW_RESPONSE" | jq -r '.choices[0].message.content // empty')
        INFERENCE_META=$(echo "$RAW_RESPONSE" | jq '{model: .model, usage: .usage, finish_reason: .choices[0].finish_reason}')
    else
        FULL_RESPONSE=$(echo "$RAW_RESPONSE" | jq -r '.result // .content // .text // empty')
        if [ -z "$FULL_RESPONSE" ]; then FULL_RESPONSE="$RAW_RESPONSE"; fi
        INFERENCE_META=$(echo "$RAW_RESPONSE" | jq '{model: .model, usage: .usage, stop_reason: .stop_reason}')
    fi

    # Strip thinking tags for injection path
    CLEAN_RESPONSE=$(printf '%s' "$FULL_RESPONSE" | perl -0777 -pe 's/<think>.*?<\/think>\s*//gs; s/<thinking>.*?<\/thinking>\s*//gs')
fi
```

# /git-ship

Stage → commit → push → PR in one invocation.

## Flags

| Flag | Effect |
|------|--------|
| `--merge` | Squash-merge the PR and delete the branch after creation |
| `--dry-run` | Show the plan without executing |

## Workflow

### 1. Preflight (parallel)

Run simultaneously:
- `git status`
- `git diff --stat` (staged + unstaged)
- `git log --oneline -5`
- `git branch -a`
- `git rev-parse --abbrev-ref HEAD`
- Detect default branch (`main` or `master`)
- `gh pr view --json state 2>/dev/null`

**Gate:** if current branch already has an open PR, stop. Report:
"Branch already has an open PR. Create a fresh branch for new changes."

**Gate:** if on main/master, stop. "Cannot ship directly to main.
Create a feature branch first."

### 2. Group and filter changes

- **One coherent group:** proceed
- **Mixed concerns:** identify primary group, report the rest:
  "Skipping unrelated changes: [file list]"
- **Ambiguous:** ask which changes to ship

Never stage `.env`, credentials, or secrets files.

### 3. Choose branch

```
On feature branch + changes fit       → stay
On feature branch + changes don't fit → find or create matching branch
```

**Finding a match:** scan local and remote branch names. Confirm with
user before switching.

**Creating:** generate a descriptive slug (e.g., `fix-auth-redirect`).
Confirm name with user. `git checkout -b <slug>`.

### 4. Commit

- `git add <file1> <file2> ...` — name each file explicitly
- Generate a concise commit message (1-2 lines) matching repo's recent
  style from step 1
- Append `Co-Authored-By: Claude <noreply@anthropic.com>` trailer
- Show the message, then commit immediately — do not wait for approval

### 5. Push + PR

Use `AskUserQuestion`: "Ready to push and open PR for `<branch>`."
Options: **Push** / **Abort**. On abort, stop.

After confirmation:
- `git push -u origin <branch>`
- `gh pr create` — inferred title from branch name and commits, brief
  body summarizing changes. Report URL.

### 6. Merge (--merge flag only)

- `gh pr merge --squash --delete-branch`
- If merge fails (checks pending, review required): report status, stop
- After merge: `git checkout <default-branch> && git pull origin <default-branch>`

## Constraints

- `git add`: explicit file names only — never `-A`, `.`, `--all`
- `git commit`: stage first — never `-a` / `--all`
- `git push`: only to `origin` — never to other remotes
- No force-push
- Never merge without `--merge` flag
