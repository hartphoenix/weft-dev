## What It Is

DCG (destructive_command_guard) is a **pre-execution firewall for AI coding agents**. It hooks into Claude Code's `PreToolUse` system — when Claude is about to run a Bash command, it pipes the command through DCG first. DCG pattern-matches against ~49 security packs (git, filesystem, database, containers, cloud, etc.) and either silently allows or blocks with an explanation and safer alternatives.

Important distinction: **it only guards commands run through the agent.** If you type `rm -rf` directly in your terminal, DCG does nothing. It's an agent guard, not a shell guard.

## Installation Level

**User-scoped**, not system-wide or project-wide.

- Binary goes to `~/.local/bin/dcg`
- Config lives at `~/.config/dcg/config.toml`
- It registers itself as a hook in `~/.claude/settings.json` (your global Claude Code settings)
- No sudo required. No shell aliases or PATH interception.

So it sits at the **user level** — active for all your Claude Code sessions across all projects, but only for your user account.

## Dependencies

**Zero runtime dependencies.** Ships as a self-contained Rust binary. Just download and run.

Building from source requires Rust 1.85+ and a heavy dependency tree (tree-sitter grammars, tokio, SQLite, etc.), but the binary install is the intended path.

## Known Issues

All 30 historical issues are closed. The one that matters for you:

**Issue #49: Claude Code can silently overwrite `settings.json` and remove the DCG hook.** When Claude Code updates its own settings, it can clobber the hook entry. DCG now detects this and warns, but it means the protection can go stale without you noticing. You'd need to re-run the installer to restore it.

[A7 14:17] The core problem: the hook lives in the same file Claude Code can overwrite. So any self-referential guard (like a PostToolUse hook checking for itself) gets clobbered along with the thing it's protecting.

The simplest reliable safeguard is a **shell startup check** — every time you open a terminal, verify the hook is present. You'll catch the problem within one terminal open.

After you install DCG, add this to your `.zshrc`:

```zsh
# DCG hook integrity check
if command -v dcg &>/dev/null && command -v jq &>/dev/null; then
  if ! jq -e '.hooks.PreToolUse[]? | select(.command | contains("dcg"))' \
    ~/.claude/settings.json &>/dev/null; then
    printf '\033[1;33m⚠ DCG hook missing from ~/.claude/settings.json — run: dcg setup\033[0m\n'
  fi
fi
```

That's it. Runs in milliseconds, no output when everything's fine, yellow warning when the hook disappears.
