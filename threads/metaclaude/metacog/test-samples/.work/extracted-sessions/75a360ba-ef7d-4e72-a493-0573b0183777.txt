## MetaClaude Security Surface

**The meta-agent has zero authority.** It's purely inference-only — receives JSON (recent turns + notepad file names), returns plain text (1-3 sentences or `[no comment]`). No CLI access, no file access, no tool use, no API calls. The model is caged by the LM Studio inference API boundary.

**Data flow:**
1. `observer.sh` (Stop hook, async) → reads transcript → curls `localhost:1234/v1/chat/completions` → writes 1-3 sentence result to staging file
2. `inject.sh` (UserPromptSubmit hook) → reads staging file → outputs as `{"additionalContext": "[MetaClaude] ..."}` → Builder Claude sees it as system context

**Output sanitization:** Thinking tags stripped (including unclosed), silence sentinel filtered, `[MetaClaude]` prefix added. All injections logged with full pipeline data.

**Write targets:** 8 dotfiles in `~/.claude/` (state/config, not code), plus log files. All writes are from your shell scripts, never from the meta-agent directly.

## Risk Analysis: Adding `localhost` to Network Allowlist

**What it enables:**
- `curl` to `localhost:1234` from inside the sandbox (observer.sh calling LM Studio)
- Any other localhost connections from sandboxed commands

**What it does NOT enable:**
- Outbound network (still blocked — localhost is not a route to the internet)
- File writes outside allowWrite paths
- Credential access
- Any new tool capabilities for Builder Claude

**Risks to consider:**

| Risk | Severity | Likelihood | Notes |
|------|----------|------------|-------|
| Builder Claude curls localhost:1234 directly | Low | Low | It has no reason to, and the response is just inference text — not executable |
| Prompt injection directs Builder to hit localhost | Low | Very low | Response is JSON chat completion — no code execution path |
