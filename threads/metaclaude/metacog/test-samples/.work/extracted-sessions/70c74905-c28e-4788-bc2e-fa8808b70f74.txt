## Finding 1: The CSS animation duration is wrong on re-render (the ring jumps)

**This is almost certainly what you saw.** It's a straightforward bug in how `--timer-duration` is set.

In [Guesses.tsx:89](src/client/Guesses.tsx#L89):
```jsx
<div className="timer" style={{ '--timer-duration': `${secsLeft}s` } as React.CSSProperties}>
```

`secsLeft` is the *current remaining time*, not the *total duration*. The CSS animation goes from full ring → empty ring over `--timer-duration` seconds. Here's what happens:

1. Phase starts. `secsLeft=10`. Animation duration = 10s. Ring depletes over 10 seconds. Correct.
2. Another player submits a guess at ~3 seconds in. Server broadcasts GUESS_STATE with `secsLeft=7.1`. Component re-renders.
3. `--timer-duration` changes to `7.1s`. The browser does NOT restart the animation — it reinterprets progress against the new duration.
4. The ring was at ~30% depleted (3s into 10s). Now the browser sees it as 3s into a 7.1s animation = **42% depleted**. The ring visually jumps forward.
5. Worse: the ring now finishes in ~4.1 more seconds instead of ~7.1. It empties out way too early.

Same issue exists in [Reveal.tsx:44](src/client/Reveal.tsx#L44).

**Fix**: `--timer-duration` should be the *initial total duration* (i.e., `GUESS_SECS` or `REVEAL_SECS`), not the current `secsLeft`. Since `<Guesses>` mounts fresh when the phase starts, setting it to the constant works — the animation starts at mount and runs for the full duration.
