[A11 18:50] Short answer: `git pull` won't silently overwrite your work. Git protects you here. What happens depends on the situation:

**If you have uncommitted changes** (like your Lobby.tsx edits right now), git will **refuse to pull** if the remote touches the same file. It'll tell you to commit or stash first.

**If you've committed your changes**, git will try to **merge** the remote changes with yours. If the same lines were edited on both sides, you get a **merge conflict** — git marks the file with both versions and asks you to pick.
