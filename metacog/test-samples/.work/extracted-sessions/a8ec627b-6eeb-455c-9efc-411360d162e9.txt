# Plan: Session Log Viewer

## Context

MetaClaude logs structured per-session JSONL to `weft-dev/meta/`, but
the format is machine-readable, not human-readable. To review sessions
— what metaclaude observed, what it injected, and the conversation
context those decisions were made in — we need a viewer that merges
Claude Code transcripts with metaclaude observation logs and presents
them as a readable chat timeline.

The PRD lists "Dashboard/viewer" as a stretch goal. This promotes it
to a real deliverable with a defined spec.

## Decisions (from user)

- **Scope:** All projects, with a selectable project filter dropdown
- **Length column:** Turns only (no token estimate)
- **File path links:** Server endpoint calls `open -R <path>` → reveals in Finder
- **Stack:** React SPA + bun HTTP server

## Step 3: Server + data layer

**`server.ts`** — Bun.serve with pattern-matched routes. Calls `Bun.build()` on startup to bundle React app into memory/dist. Routes:

- `GET /api/projects` — scans `~/.claude/projects/`, returns `[{id, name}]`
  - Name derived from dir: split on `-Documents-GitHub-`, take suffix
- `GET /api/sessions?project=<id>` — returns session summaries
  - Scans transcript JSONL files (first ~20 lines each for metadata)
  - Joins with metaclaude logs by UUID prefix match (meta log filename contains first 8 chars of transcript UUID — e.g., `meta/2026-03-09_9bfb385c.jsonl` matches transcript `9bfb385c-f10c-415b-85b8-aafd6dfcbaf3.jsonl`)
  - Cached in memory; `GET /api/refresh` rebuilds cache
- `GET /api/session/:id` — full parsed conversation
  - Returns `{turns[], toolResults{}, meta[]}`
  - Turns = user + assistant entries; tool_result user entries skipped (paired by ID instead). Note: Claude Code encodes tool results as `type: "user"` entries whose `message.content` is an array of `{type: "tool_result"}` objects — these are not human messages.
- `POST /api/reveal` — body `{path}`, calls `Bun.spawn(["open", "-R", path])`
- `GET /*` — serves static React build

**`lib/parse-transcript.ts`** — JSONL parser. Two modes:
- Summary mode: read first 20 lines, extract sessionId, timestamp, cwd, gitBranch
- Full mode: parse all lines, filter to user/assistant, normalize content to arrays, build tool_result pairing map

**`lib/parse-meta.ts`** — Parse metaclaude JSONL. Build `Map<uuid_prefix, MetaLogData>`.

**`lib/session-index.ts`** — Orchestrates: discover projects → scan transcripts (summary mode) → join meta logs → cache. Returns sorted session list.
