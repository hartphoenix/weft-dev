[A3 17:29] Good news and bad news here. There are hooks for tool-level failures, but the 500 errors you're hitting are a different beast.

## Two different failure modes

**Tool failures** (Edit can't find the string, Bash command errors, etc.) — Claude Code has a `PostToolUseFailure` hook that fires after these. You can use it to inject recovery guidance.

**API 500 errors** (Anthropic's server returns a 500) — these kill the agent's turn entirely. The agent doesn't get a chance to "decide" what to do because the API call itself failed. Claude Code surfaces the error and waits for you to say "continue." There's no hook for this because it happens at the transport layer, before any agent logic runs.

## What you CAN do

### 1. `PostToolUseFailure` hook for tool-level errors

This catches things like failed edits, permission errors, file-not-found, etc. and feeds recovery context back to the agent:

```json
// .claude/settings.json
{
  "hooks": {
    "PostToolUseFailure": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/error-recovery.sh"
          }
        ]
      }
    ]
  }
}
```

The hook receives JSON with `tool_name`, `tool_input`, and `error`, and can return `additionalContext` that tells the agent what to check before retrying.

## Hook placement: user-level, not `GitHub/.claude/`

Hooks don't cascade from parent project directories to child projects. When you open a session in `maestro/`, Claude Code reads:
- `~/.claude/settings.json` (user-level — always)
- `maestro/.claude/settings.json` (project-level)

It does **not** read `GitHub/.claude/settings.json` unless `GitHub/` itself is the workspace root. So putting the hook in `GitHub/.claude/` would only work for sessions opened at that directory level.

The right place is **`~/.claude/settings.json`** — where your `dcg` PreToolUse hook already lives. One definition, applies everywhere. We can add it alongside the existing hook when you're ready for step 2.
